# DOCX_TEMPLATE_MAPPING.md

## Назначение

Документ фиксирует карту соответствия между переменными DOCX-шаблона и данными системы `permit-system` для наряда-допуска `Permit`.

Карта нужна для проверки реального DOCX-шаблона организации перед загрузкой в `DocumentTemplate` и генерацией через основной сервис `generate_permit_docx`.

> Важно: этот файл описывает соглашение по шаблонам и проверочный список. Он не меняет модели, бизнес-логику или сервис генерации.

## Рекомендуемый подход

- Для новых русскоязычных шаблонов используйте русские переменные верхнего уровня: `{{ номер_наряда }}`, `{{ место_работ }}`, `{{ опасности }}`.
- Существующие английские переменные остаются совместимыми и не удаляются, чтобы старые шаблоны продолжали работать.
- Даты в русских алиасах отдаются в формате `ДД.ММ.ГГГГ`.
- Списки опасностей и мер безопасности отдаются готовым текстом через запятую.
- Справочники и ручные поля используются совместно: справочники остаются для типовых значений, а ручные поля уточняют нестандартные случаи.
- Пустые значения, например отсутствие оборудования или дополнительных условий, отдаются как пустая строка.

## Основная карта русских переменных DOCX-шаблона

| Переменная шаблона | Источник данных в системе | Модель/поле | Обязательное поле | Пример значения | Комментарий |
| --- | --- | --- | --- | --- | --- |
| `номер_наряда` | Номер наряда-допуска | `Permit.number` | да | `ND-2026-001` | Основной номер Permit. |
| `статус_наряда` | Человекочитаемый статус lifecycle | `Permit.get_status_display()` | да | `Draft` | Для русских названий статусов нужен отдельный перевод choices/UI. |
| `участок` | Участок работ | `Permit.work_area.name` | да | `Цех 1` | Готовое текстовое значение участка. |
| `оборудование` | Оборудование | `Permit.equipment.name` | нет | `Насос H-101` | Если оборудование не выбрано, возвращается пустая строка. |
| `вид_работ` | Вид работ | `Permit.work_type.name` | да | `Огневые работы` | Готовое текстовое значение вида работ. |
| `место_работ` | Место выполнения работ | `Permit.work_location` | да | `Отм. +6.000, площадка обслуживания` | Короткое текстовое поле. |
| `описание_работ` | Описание работ | `Permit.work_description` | да | `Ремонт запорной арматуры на линии подачи` | Длинный текст; проверить переносы в DOCX. |
| `характер_работ` | Ручное описание характера нестандартных работ | `Permit.work_nature_text` | нет | `Осмотр и ремонт запорной арматуры` | Используется вместе со справочным `WorkType`, не заменяет справочник. |
| `дополнительные_условия` | Дополнительные условия / уточнения | `Permit.additional_conditions` | нет | `Работы выполнять после инструктажа` | Для условий, которых нет в справочниках. |
| `дополнительные_меры_безопасности` | Дополнительные меры безопасности / примечания | `Permit.additional_safety_notes` | нет | `Использовать диэлектрические перчатки` | Дополняет справочник `SafetyMeasure`. |
| `дата_начала` | Дата начала работ | `Permit.work_starts_at` | да | `13.06.2026` | Формат `ДД.ММ.ГГГГ`. |
| `дата_окончания` | Дата окончания работ | `Permit.work_ends_at` | да | `13.06.2026` | Формат `ДД.ММ.ГГГГ`. |
| `дата_создания` | Дата создания наряда | `Permit.created_at` | да | `12.06.2026` | Формат `ДД.ММ.ГГГГ`. |
| `ответственный_руководитель` | Ответственный руководитель | `Permit.responsible_manager_text` или `Permit.responsible_manager` | да | `Иванов И.И.` | Если ручное поле заполнено, используется оно; иначе ФИО/username пользователя. |
| `производитель_работ` | Производитель работ / непосредственный руководитель | `Permit.work_producer_text` или `Permit.work_supervisor` | да | `Петров П.П.` | Если ручное поле заполнено, используется оно; иначе ФИО/username пользователя. |
| `создал_пользователь` | Автор наряда | `Permit.created_by` | да | `operator` | Используется ФИО пользователя, если заполнено; иначе username. |
| `опасности` | Опасности | `Permit.hazards` → `Hazard.name` | нет | `Высота, Электрическое напряжение` | Готовый текст через запятую. |
| `меры_безопасности` | Меры безопасности | `Permit.safety_measures` → `SafetyMeasure.name` | нет | `Отключить питание, Оформить ограждение` | Готовый текст через запятую. |
| `шаблон_документа` | Название DOCX-шаблона | `DocumentTemplate.name` | да | `Наряд-допуск v1` | Передаётся из выбранного шаблона генерации. |
| `версия_шаблона` | Версия DOCX-шаблона | `DocumentTemplate.version` | да | `2026-01` | Передаётся из выбранного шаблона генерации. |

## Что нужно проверить на реальном шаблоне

Перед использованием реального DOCX-шаблона организации нужно проверить:

- [ ] Названия переменных совпадают с фактическим контекстом `generate_permit_docx`.
- [ ] В шаблоне нет опечаток в именах переменных.
- [ ] Русские переменные вставлены без пробелов внутри имени, например `{{ номер_наряда }}`.
- [ ] Таблицы DOCX корректно сохраняют структуру после генерации.
- [ ] Перенос строк в длинных текстах выглядит читаемо.
- [ ] Поля подписей расположены корректно и не съезжают после генерации.
- [ ] Даты отображаются в формате `ДД.ММ.ГГГГ`.
- [ ] Списки опасностей отображаются в согласованном формате.
- [ ] Списки мер безопасности отображаются в согласованном формате.
- [ ] Длинные тексты в `описание_работ` не ломают таблицы и страницы.
- [ ] Форматирование после генерации соответствует исходному шаблону: шрифты, интервалы, границы таблиц, выравнивание.
- [ ] Пустое оборудование отображается корректно, если наряд создаётся без `Equipment`.
- [ ] ФИО/username пользователей отображаются в ожидаемом виде.
- [ ] Готовый DOCX открывается в Microsoft Word/LibreOffice без предупреждений о повреждении файла.

## Рекомендации по первому реальному шаблону

1. Начать с минимального шаблона, где есть только основные поля: `номер_наряда`, `статус_наряда`, `участок`, `вид_работ`, `дата_начала`, `дата_окончания`, `место_работ`.
2. Проверить генерацию и скачивание DOCX.
3. Добавить пользователей и подписи.
4. Добавить опасности и меры безопасности.
5. Проверить длинные тексты и таблицы.
6. Только после этого переносить полный производственный шаблон.
