# DOCX_TEMPLATE_MAPPING.md

## Назначение
Карта переменных DOCX-шаблона для `permit-system`. Новые шаблоны должны использовать транслит-переменные; старые английские `permit.*` остаются только для совместимости.

> Предупреждение: **Не используйте кириллицу внутри {{ ... }}.** Пишите `{{ nomer_naryada }}`, а не `{{ номер_наряда }}`.

## Почему транслит
`docxtpl` использует Jinja-переменные внутри DOCX. Word/LibreOffice могут разбивать текст переменной на внутренние фрагменты. Латиница, цифры и подчёркивания устойчивее и проще проверяются тестами.

## Основные переменные
| Переменная | Русское описание |
| --- | --- |
| `{{ nomer_naryada }}` | Номер наряда |
| `{{ status_naryada }}` | Статус наряда по-русски |
| `{{ uchastok }}` | Участок работ |
| `{{ oborudovanie }}` | Оборудование |
| `{{ vid_rabot }}` | Вид работ |
| `{{ mesto_rabot }}` | Место работ |
| `{{ opisanie_rabot }}` | Описание работ |
| `{{ harakter_rabot }}` | Характер работ, ручное поле |
| `{{ data_nachala }}` | Дата начала работ в формате ДД.ММ.ГГГГ |
| `{{ data_okonchaniya }}` | Дата окончания работ в формате ДД.ММ.ГГГГ |
| `{{ data_sozdaniya }}` | Дата создания наряда |
| `{{ otvetstvennyy_rukovoditel }}` | Первый ответственный руководитель или fallback из User/ручного поля |
| `{{ proizvoditel_rabot }}` | Первый производитель работ или fallback из User/ручного поля |
| `{{ sozdal_polzovatel }}` | Пользователь, создавший наряд |
| `{{ opasnosti }}` | Опасности через запятую |
| `{{ mery_bezopasnosti }}` | Меры безопасности через запятую |
| `{{ dopolnitelnye_usloviya }}` | Дополнительные условия |
| `{{ dopolnitelnye_mery_bezopasnosti }}` | Дополнительные меры безопасности |
| `{{ shablon_dokumenta }}` | Название шаблона |
| `{{ versiya_shablona }}` | Версия шаблона |

## Участники
| Переменная | Русское описание |
| --- | --- |
| `{{ otvetstvennye_rukovoditeli }}` | Все ответственные руководители, по одному на строку |
| `{{ proizvoditeli_rabot }}` | Все производители работ |
| `{{ ispolniteli }}` | Исполнители работ |
| `{{ chleny_brigady }}` | Члены бригады |
| `{{ dopuskayushchie }}` | Допускающие |
| `{{ nablyudayushchie }}` | Наблюдающие |
| `{{ prochie_uchastniki }}` | Прочие участники |
| `{{ uchastniki_rabot }}` | Общий список участников по строкам |

## Примеры

### Простой шаблон
```text
Наряд-допуск № {{ nomer_naryada }}
Статус: {{ status_naryada }}
Место работ: {{ mesto_rabot }}
Описание: {{ opisanie_rabot }}
```

### Участники
```text
Ответственный руководитель:
{{ otvetstvennye_rukovoditeli }}

Бригада:
{{ uchastniki_rabot }}
```

### Даты
```text
Работы выполнить с {{ data_nachala }} по {{ data_okonchaniya }}.
Наряд создан: {{ data_sozdaniya }}.
```

### Меры безопасности
```text
Опасности: {{ opasnosti }}
Меры безопасности: {{ mery_bezopasnosti }}
Дополнительно: {{ dopolnitelnye_mery_bezopasnosti }}
```


## Нерекомендуемые кириллические переменные
Кириллические имена переменных внутри `{{ ... }}` не рекомендуются для новых шаблонов. Если в старом файле есть `{{ номер_наряда }}` или похожие имена, замените их на транслит-варианты из таблицы выше.

## Проверка шаблона
Загрузите DOCX в `DocumentTemplate`, отметьте тип `permit` и используйте проверку шаблона на demo-данных в Django admin. Проверка помогает найти ошибки до генерации реального `GeneratedDocument`.

## Частые ошибки
- `No active docx template` — нет активного `DocumentTemplate` с `document_type="permit"`. Загрузите и активируйте шаблон.
- `unexpected '}'` — лишняя закрывающая скобка или повреждённая Jinja-конструкция. Проверьте все `{{ ... }}`.
- Переменная не подставилась — имя переменной написано с ошибкой, используется кириллица или переменной нет в контексте.
- Дата не отображается — дата отсутствует в данных или в шаблоне использована не транслит-переменная. Используйте `{{ data_nachala }}`, `{{ data_okonchaniya }}`, `{{ data_sozdaniya }}`.
