# PROJECT_STATE.md

## Проект

`permit-system` — внутренняя веб-система для создания, согласования, хранения и печати нарядов-допусков по DOCX-шаблонам.

## Цель системы

Система должна позволять сотрудникам организации:

- создавать наряды-допуски;
- заполнять сведения о месте работ, оборудовании, виде работ, опасностях и мерах безопасности;
- согласовывать наряды по ролям и workflow;
- хранить историю действий и аудита;
- генерировать документы по DOCX-шаблонам;
- скачивать готовые DOCX-документы;
- управлять шаблонами документов через веб-интерфейс.

## Зафиксированный контекст реализации

По проектному контексту уже выполнены следующие части:

- создан начальный Django-проект;
- подготовлена Docker/Docker Compose заготовка;
- реализованы пользователи и роли;
- заведены роли `operator`, `master`, `chief`, `admin`;
- реализованы модели `Permit`, `ApprovalAction`, `AuditLog`, `DocumentTemplate`, `GeneratedDocument`;
- реализован workflow согласований;
- реализованы сервисы согласования;
- реализована генерация DOCX;
- создан минимальный веб-интерфейс для `Permit`;
- добавлены действия согласования через интерфейс;
- добавлено скачивание DOCX;
- заведены справочники `WorkArea`, `Equipment`, `WorkType`, `Hazard`, `SafetyMeasure`;
- создан интерфейс управления DOCX-шаблонами;
- убрана обязательная проверка screenshot в Codex Cloud.

## Фактическое состояние текущего checkout

В текущем checkout присутствует Django-проект `permit-system`: есть `manage.py`, `requirements.txt`, `requirements-dev.txt`, `docker-compose.yml`, настройки `permit_system/` и приложения `users`, `permits`, `approvals`, `documents`, `audit`.

Обязательные Django-проверки нужно запускать из корня репозитория, где находится `manage.py`. Если зависимости не установлены, сначала используйте `python -m pip install -r requirements-dev.txt` или Docker Compose согласно `README.md`.

## Стабильные архитектурные решения

- Основной backend-фреймворк: Django.
- Основная база данных: PostgreSQL.
- Среда запуска: Docker/Docker Compose.
- UI: Django templates.
- Генерация документов: `docxtpl` / `python-docx`.
- Текущий frontend-подход: без React.

## Ключевые сущности

- `Permit` — наряд-допуск.
- `ApprovalAction` — действие согласования.
- `AuditLog` — журнал аудита значимых действий.
- `DocumentTemplate` — DOCX-шаблон документа; единственный источник правды для шаблонов.
- `GeneratedDocument` — сгенерированный документ; единственный источник правды для готовых документов.
- `WorkArea` — место/зона работ.
- `Equipment` — оборудование.
- `WorkType` — вид работ.
- `Hazard` — опасность.
- `SafetyMeasure` — мера безопасности.

## Ключевые роли

- `operator` — создаёт и ведёт наряды в рамках доступных действий.
- `master` — участвует в согласовании/проверке нарядов по workflow.
- `chief` — участвует в согласовании/утверждении нарядов по workflow.
- `admin` — управляет системой, пользователями, справочниками и шаблонами в рамках административных полномочий.
