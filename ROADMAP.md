# ROADMAP.md

## Назначение
Roadmap фиксирует фактический прогресс и дальнейший план развития `permit-system`.

## Принципы
- Не менять workflow согласований без отдельного решения.
- Не дублировать модели, формы, views, templates и сервисы.
- Сохранять `DocumentTemplate` и `GeneratedDocument` единственными источниками правды для шаблонов и готовых документов.
- Развивать UI на Django templates без React и без внешних CDN.
- Документацию обновлять по факту: planned-функции помечать как pending.

## Выполнено
- [x] Базовый Django-проект, Docker Compose, SQLite fallback для локальных проверок.
- [x] Роли `operator`, `master`, `chief`, `admin`.
- [x] MVP workflow наряда: создание, редактирование в разрешённых статусах, отправка, согласование, возврат, утверждение, отклонение, закрытие.
- [x] Справочники работ, опасностей и мер безопасности.
- [x] Справочник персонала `PersonnelGroup`/`Personnel` и участники наряда `PermitParticipant`.
- [x] Ручной ввод участника, ответственных и дополнительных условий.
- [x] DOCX-генерация, скачивание DOCX, проверка шаблона на demo-данных.
- [x] Безопасные транслит-переменные DOCX.
- [x] Demo setup через `seed_demo_data`.
- [x] Базовый аудит и история согласований.
- [x] Подготовительный PDF-слой, выключенный по умолчанию.
- [x] Проверка отсутствия внешних CDN в текущем UI.
- [x] FAQ: `FAQ.md`, страница `/faq/`, ссылка в меню и тесты доступности.

## Ближайшие этапы

### 1. Улучшение UX
- Подключить autocomplete персонала к существующему `/personnel/search/?q=` без CDN.
- Улучшить подсказки в форме наряда и блоке участников.
- Добавить более удобный UI управления DOCX-шаблонами.

### 2. Поиск и фильтры
- Фильтры нарядов по статусу, участку, оборудованию, виду работ, датам и автору.
- Поиск по номеру и описанию.
- Представление «ожидает моего действия» для ролей.

### 3. Официальный DOCX-шаблон
- Получить реальный шаблон организации.
- Перенести поля на транслит-переменные.
- Проверить шаблон через admin demo-check и на реальном наряде.

### 4. Production deployment
- Финализировать `.env`, `ALLOWED_HOSTS`, `CSRF_TRUSTED_ORIGINS`, `STATIC_ROOT`, `MEDIA_ROOT`.
- Настроить PostgreSQL, static/media volumes, reverse proxy и TLS.
- Подготовить инструкцию обновления и отката.

### 5. Резервное копирование
- Настроить backup PostgreSQL и media.
- Проверить восстановление до пилота.
- Документировать RPO/RTO и ответственных.

### 6. PDF через LibreOffice, если нужно
- Решить, нужен ли PDF на пилоте.
- Установить и проверить LibreOffice/`soffice` на сервере.
- Добавить пользовательский сценарий PDF-экспорта поверх существующего сервиса.

### 7. Аудит безопасности
- Проверить настройки production Django.
- Проверить роли, доступ к admin, static/media, секреты и логи.
- Проверить, что документы не доступны без авторизации там, где это требуется.

### 8. Тестирование пользователями
- Провести демонстрации операторам, мастерам, начальникам и администраторам.
- Сверить названия полей, роли участников, DOCX-переменные и FAQ.
- Приоритизировать доработки по обратной связи.

## Устаревшие или отложенные задачи
- React/SPA не планируется на текущем этапе.
- Внешние CDN не использовать.
- Кириллические DOCX-переменные не развивать; для новых шаблонов использовать транслит.
